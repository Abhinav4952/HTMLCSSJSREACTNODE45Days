### `compoundSpecificityParts(selector)`

- Accepts trimmed strings using **only** this lab grammar:
  - Optional **one** type selector at the very beginning: lowercase letters `[a-z]+` (e.g. `div`, `article`).
  - Then zero or more tokens, each either `#id` or `.class`, immediately concatenated (no spaces).
  - `id` / `class` identifiers match: first char `[A-Za-z_]`, then optional `[A-Za-z0-9_-]*`.
- Returns `{ ids, classes, types }` where:
  - `types` is `1` if a type selector was present, else `0`.
  - `ids` counts `#...` tokens.
  - `classes` counts `....` tokens.
- Returns `null` for invalid input: non-strings, empty/whitespace-only strings, unsupported characters (`:`, spaces, combinators, etc.), malformed `#` / `.`, uppercase type tags like `DIV`.

### `compareSpecificityParts(a, b)`

- `a` and `b` are objects like `{ ids, classes, types }` with finite non-negative integers.
- Compare like CSS: higher `ids` wins; if tied, higher `classes`; if tied, higher `types`.
- Return `1` if `a` wins, `-1` if `b` wins, `0` on full tie.

### `effectiveNestedOpacity(parentOpacity, childOpacity)`

- Return `parentOpacity * childOpacity` when both are finite numbers in **`[0, 1]`** (inclusive).
- Otherwise return `NaN`.

### `blendRgbOver(bottomRgb, topRgb, topAlpha)`

- `bottomRgb` and `topRgb` must each be length-3 arrays of **integers** in `0..255` inclusive (non-integers → `null`).
- `topAlpha` must be finite and in `0..1` inclusive.
- Return `[r,g,b]` integers where each channel is `Math.round(top*alpha + bottom*(1-alpha))`, then clamped into `0..255` (rounding should already land in range for valid inputs, but clamp defensively).
- Return `null` for invalid shapes/values (`NaN` channels, alpha out of range, non-arrays, wrong lengths).
