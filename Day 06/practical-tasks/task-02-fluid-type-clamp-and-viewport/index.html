Fluid typography is bounded improvisation

You are not trying to make text infinitely large on ultra-wide monitors. You are choosing a readable band that tracks viewport width gently.

Start from humans: short lines are easier to read, so max-width in ch still matters even when font sizes swell. Then pick minima that survive phones in portrait mode.

“Slope without a floor is just chaos with fonts.”

When you finish the CSS TODOs, resize the window slowly: the headline should glide, not stair-step, because clamp() is continuous across widths—unlike breakpoint-only font swaps.

Verify computed font sizes at 320px and 1280px in DevTools.
