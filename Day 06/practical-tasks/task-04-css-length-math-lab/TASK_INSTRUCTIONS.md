# Task 04 — CSS Length Math Lab (Vitest)

## Summary

Implement a tiny **pure-math utilities** module that mirrors the mental math you do when converting `rem` → `px`, clamping fluid values, and computing nested `em` font-size products. This is **not** a full CSS parser; it encodes a handful of **contracts** you will rely on when debugging layout in DevTools.

**Challenge level:** edge cases (`NaN`, backwards bounds, weird whitespace) are intentionally tested.

### Public API (required)

1. `clampNumber(value, min, max)`
   - If any argument is `NaN`, return `NaN`.
   - If `max < min`, throw `RangeError` with message exactly: `max < min`.
   - Otherwise return `value` bounded to `[min, max]` (inclusive).

2. `parsePx(input)`
   - Accepts strings like `"12px"`, `" 12.5px "` (trim whitespace).
   - Returns a finite **number** of px.
   - Returns `null` for invalid input (wrong unit, missing number, empty string, non-string).

3. `parseRem(input)`
   - Same trimming rules; accepts `"1.25rem"`.
   - Returns numeric **rem** factor (not px).
   - Returns `null` if invalid.

4. `remToPx(rem, rootPx = 16)`
   - Returns `rem * rootPx`.
   - If `rem` is not finite OR `rootPx` is not finite, return `NaN`.
   - If `rootPx <= 0`, return `NaN` (font size can’t be non-positive).

5. `nestedEmFontPx(emFactors, rootPx = 16)`
   - `emFactors` is an array of positive finite numbers representing chained `em` font-size multipliers on nested wrappers.
   - Returns `rootPx * product(emFactors)`.
   - If array is **empty**, return `rootPx` (identity).
   - If any factor is not finite or `<= 0`, return `NaN`.
   - If `rootPx` not finite or `<= 0`, return `NaN`.

6. `vwContributionPx(vw, viewportWidthPx)`
   - Returns `vw * viewportWidthPx / 100`.
   - If either not finite, or `viewportWidthPx < 0`, return `NaN`.
