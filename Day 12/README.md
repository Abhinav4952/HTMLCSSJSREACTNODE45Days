# Day 12 — HTML5 Semantics, Containers, and Rich Inputs

## Learning objectives

- Choose **semantic sectioning elements** (`header`, `nav`, `main`, `aside`, `footer`, `section`, `article`) for real page skeletons—not `div` soup.
- Use **HTML5 input types** (`email`, `url`, `number`, `range`, `date`, etc.) where they improve mobile keyboards and basic validation.
- Apply **constraint validation** attributes (`required`, `min`, `max`, `step`, `pattern`, `minlength`, `maxlength`) with clear, accessible error UX.
- Connect semantics to **accessibility basics**: skip links, landmark roles implied by elements, labels for every control.

## Prerequisites

- Days **01–04** (page structure, forms foundation) and **Day 11** (cascade thinking helps when styling validation states).
