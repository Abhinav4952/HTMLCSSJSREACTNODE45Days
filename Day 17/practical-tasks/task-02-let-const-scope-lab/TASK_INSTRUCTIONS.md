# Task 02 — `let` / `const` / `var` Scope Lab (Vitest)

## Summary

Encode four small “scope lessons” as functions with **locked outputs**—including the classic `var` loop closure trap.

## Learning goals

- Explain why `var` + closures surprises beginners.
- Use `let` to get per-iteration bindings.

## Provided files

- `package.json`, `.npmrc`
- `src/scopeLab.js`
- `tests/scopeLab.test.js`

## Prerequisites

- Day 17 theory: `const`/`let`/`var`.

## What you will implement

Complete every `// TODO(Day17-task02):` in `src/scopeLab.js`.

### `varLoopClosureMappedValues()`

Create **three** functions in a loop using `var i` from `0` to `2` (inclusive), push `() => i` each time, then return `fns.map((fn) => fn())`.

Expected: `[3, 3, 3]`.

### `letLoopClosureMappedValues()`

Same structure, but use `let i`.

Expected: `[0, 1, 2]`.

### `blockShadowValue()`

Return the inner value from this pattern:

```js
let x = 1;
{
  let x = 2;
  return x;
}
```

Expected: `2`.

### `constObjectPropertyMutation()`

Start with `const o = { n: 3 }`, set `o.n = 7`, return `o.n`.

Expected: `7`.
